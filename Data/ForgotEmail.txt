-Api localhost:8080/api/user/forgot, server gửi mail xác thực đến người dùng => trả về
{
    "email": "**@gmail.com",
    "password": null,
    "token": "",
    "status": "Send mail reset pass success!"
}
-Api xác thực: http://localhost:8080/api/user/forgot/confirm?token=** => trả về
{
    "user": {
        **
    },
    "token": *token sẽ được trả về*,
    "status": "Allow user change pass."
}
*Khi người dùng chưa xác thực email => không cho phép đổi mật khẩu.
Thời gian cho phép đổi mật khẩu tính từ thời điểm server gửi email: 5 phút.
Đổi mật khẩu:
Api: localhost:8080/api/user/forgot/changePassword
front-end gửi back-end với định dạng:
{
    "email": "email người dùng", // ẩn trên front-end
    "password": "password mới",
    "token": "token đc trả về từ verify"
}
Khi đổi thành công, server trả về:
{
    "user": null,
    "token": null,
    "status": "User update password successful."
}

-Api localhost:8080/api/user/register/resend : resend token confirm
Front-end truyền vào với định dạng:
{
    "email": "Email người dùng cần confirm"
}
trả về
{
    "user": null,
    "token": "",
    "status": "Resend Email success."
}
